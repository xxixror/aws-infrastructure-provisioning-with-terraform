# AWS Infrastructure Provisioning with Terraform

## Overview
This project focuses on provisioning **AWS infrastructure using Terraform**
as **Infrastructure as Code (IaC)**.

The goal of the project is to understand how cloud infrastructure can be
**defined, deployed, and managed through code**, while applying
**basic security considerations** such as access control and network separation.

---

## Scope
- AWS infrastructure provisioning using Terraform
- Basic network architecture design
- IAM configuration for access control
- Reproducible infrastructure deployment using IaC

---

## Objectives
- Learn Infrastructure as Code (IaC) concepts
- Reduce manual cloud configuration through automation
- Understand basic cloud security concepts such as access control and network separation

---

## Environment & Tools
- **Cloud Provider**: AWS
- **IaC Tool**: Terraform
- **Identity & Access Management**: IAM
- **Networking**: Virtual Private Cloud (VPC)

---

## Project Description
Terraform is used to define and provision AWS resources through declarative
configuration files. Infrastructure components such as networking and access
control are managed as code, enabling consistent and repeatable deployments.

By using IaC, this project demonstrates how cloud environments can be
provisioned in a structured and automated manner, reducing configuration drift
and improving maintainability.
