# AWS Infrastructure Provisioning with Terraform

## Overview
This project focuses on provisioning **AWS infrastructure using Terraform**
as **Infrastructure as Code (IaC)**.

The goal of the project is to understand how cloud infrastructure can be
**defined, deployed, and managed through code**, while incorporating
**high availability and basic security controls**.

---

## Scope
- AWS infrastructure provisioning using Terraform
- High availability architecture using Auto Scaling and Load Balancing
- Basic network architecture design using VPC
- IAM configuration for access control
- Web application protection using AWS WAF
- Reproducible infrastructure deployment using IaC

---

## Objectives
- Learn Infrastructure as Code (IaC) concepts
- Reduce manual cloud configuration through automation
- Understand high availability design using Auto Scaling and Load Balancers
- Apply basic cloud security concepts such as access control, network separation,
  and web attack mitigation

---

## Environment & Tools
- **Cloud Provider**: AWS
- **IaC Tool**: Terraform
- **Identity & Access Management**: IAM
- **Networking**: Virtual Private Cloud (VPC)
- **High Availability**:
  - Auto Scaling Group
  - Load Balancer
- **Security**:
  - AWS WAF

---

## Project Description
Terraform is used to define and provision AWS resources through declarative
configuration files.

The infrastructure is designed with **high availability** in mind by using
Auto Scaling Groups and a Load Balancer to distribute traffic and automatically
adjust capacity based on demand.

Basic security principles are incorporated into the design, including:
- IAM-based access control
- Network isolation using VPC
- Web request filtering using AWS WAF to mitigate common attacks such as
  SQL injection and brute-force attempts

By managing the entire infrastructure as code, this project demonstrates how
cloud environments can be deployed consistently, securely, and reproducibly.
